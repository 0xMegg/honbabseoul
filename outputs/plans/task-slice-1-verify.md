### Schema-shape checks (against the live DB)
8. **Table exists with right shape.** `psql "$DATABASE_URL" -c "\d restaurants"` — must list **all 16 columns** with correct types:
   ```
   id              | uuid                     | not null default gen_random_uuid()
   name_ja         | text                     |
   name_ko         | text                     |
   address_ja      | text                     |
   address_ko      | text                     |
   latitude        | double precision         |
   longitude       | double precision         |
   price_range     | price_range              |
   status          | restaurant_status        | not null default 'pending'::restaurant_status
   is_solo_default | boolean                  | not null default true
   has_jp_menu     | boolean                  | not null default false
   is_late_night   | boolean                  | not null default false
   naver_url       | text                     |
   photo_url       | text                     |
   created_at      | timestamp with time zone | not null default now()
   updated_at      | timestamp with time zone | not null default now()
   ```
9. **Enums exist with right values.** `psql "$DATABASE_URL" -c "\dT+ restaurant_status"` lists `pending, approved, rejected`. `psql "$DATABASE_URL" -c "\dT+ price_range"` lists `low, mid, high`.
10. **RLS is enabled.** `psql "$DATABASE_URL" -c "select tablename, rowsecurity from pg_tables where schemaname='public' and tablename='restaurants'"` returns `restaurants | t`.
11. **Both anon policies exist.** `psql "$DATABASE_URL" -c "select polname, polcmd from pg_policy where polrelid='public.restaurants'::regclass"` returns at least:
    ```
    anon read approved        | r  (SELECT)
    anon insert pending only  | a  (INSERT)
    ```
    No UPDATE / DELETE policies should be present (anything beyond the above means a leak risk).
12. **Triggers exist.** `psql "$DATABASE_URL" -c "select tgname from pg_trigger where tgrelid='public.restaurants'::regclass and not tgisinternal"` returns BOTH `restaurants_set_updated_at` and `restaurants_force_pending_for_anon`.
13. **Indexes exist.** `psql "$DATABASE_URL" -c "\di restaurants_*"` lists both `restaurants_status_idx` and `restaurants_status_solo_idx` plus the implicit `restaurants_pkey`.

### RLS behavior probes (the cardinal contract — Reviewer reruns these)
14. **Anon SELECT respects `status='approved'`.** With the table empty, the response is `[]`. Probe:
    ```
    curl -s -H "apikey: $NEXT_PUBLIC_SUPABASE_ANON_KEY" -H "Authorization: Bearer $NEXT_PUBLIC_SUPABASE_ANON_KEY" "$NEXT_PUBLIC_SUPABASE_URL/rest/v1/restaurants?select=id,status"
    ```
    Expected: `[]`. Record the exact response body.
15. **Anon INSERT silently downgrades `status='approved'` to `pending`.** Probe:
    ```
    curl -s -X POST -H "apikey: $NEXT_PUBLIC_SUPABASE_ANON_KEY" -H "Authorization: Bearer $NEXT_PUBLIC_SUPABASE_ANON_KEY" -H "Content-Type: application/json" -H "Prefer: return=representation" "$NEXT_PUBLIC_SUPABASE_URL/rest/v1/restaurants" -d '{"name_ja":"_rls_smoke_review_","name_ko":"_RLS_리뷰스모크_","status":"approved","naver_url":"https://map.naver.com/p/rls-review","is_solo_default":true,"has_jp_menu":false,"is_late_night":false,"price_range":"low","latitude":37.5666,"longitude":126.9784}'
    ```
    Expected: response JSON contains `"status":"pending"` (NOT `"approved"`). The anon-key INSERT itself succeeds because the with-check policy reads the post-trigger value, which is now `'pending'`.
    Verify via service-role psql:
    ```
    psql "$DATABASE_URL" -c "select status from restaurants where name_ja='_rls_smoke_review_'"
    ```
    Expected: `pending`.
16. **Anon SELECT still hides the freshly-inserted pending row.** Re-run the GET from step 14. Expected: still `[]` (the inserted row's `status='pending'` does NOT match the anon SELECT policy).
17. **Anon UPDATE is denied.** Probe:
    ```
    curl -s -X PATCH -H "apikey: $NEXT_PUBLIC_SUPABASE_ANON_KEY" -H "Authorization: Bearer $NEXT_PUBLIC_SUPABASE_ANON_KEY" -H "Content-Type: application/json" -H "Prefer: return=representation" "$NEXT_PUBLIC_SUPABASE_URL/rest/v1/restaurants?name_ja=eq._rls_smoke_review_" -d '{"status":"approved"}'
    ```
    Expected: empty array `[]` (no rows updated — the row is RLS-hidden from anon SELECT, and there's no UPDATE policy). Record the exact response.
18. **Anon DELETE is denied.** Similar curl with `-X DELETE`. Expected: empty array `[]` or 401/403.
19. **Cleanup smoke row.** `psql "$DATABASE_URL" -c "delete from restaurants where name_ja='_rls_smoke_review_'"` then `psql "$DATABASE_URL" -c "select count(*) from restaurants"` → `0`. The Developer ran an equivalent during Phase I; the Reviewer reruns to ensure no leftover before Slice 2.
