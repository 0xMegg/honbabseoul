## Retrospective

**One-line summary**: Released v3.0, then immediately fixed 3 real-world issues discovered by using it ourselves.

**What went well**:

- The "build -> use -> fix" cycle was short. 3 fix commits within 3 hours after the v3 release.
- The Reviewer workflow commit order issue was a bug that could never be found without actually running it. The power of real-world usage.
- Slash commands significantly reduced role-switching friction. A single word `/review` enters Reviewer mode.

**Lessons learned**:

- **Order** in workflows is really important. If commit runs before report, files naturally get missed — but it's easy to overlook in documentation alone.
- When adding new features, always verify that **references are connected** in existing workflows (the verify.md omission case).
