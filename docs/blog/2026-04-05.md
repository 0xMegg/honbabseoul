# 2026-04-05 — v4.0.0 Upgrade: BLOCK/WARN Hooks + Automation Scripts

> A day of splitting the hook system by severity and adding Task/Epic automation scripts to make the 3-Role Workflow executable with a single command.

---

## Background

Up through v3, post-edit hooks used simple binary PASS/FAIL judgments. Through real-world usage on DiveBase, the need arose to distinguish between things requiring immediate fixes (BLOCK) — like lint errors — and things that can be left to the reviewer (WARN). Additionally, there was a desire to automate the repetitive manual invocation of `/plan` -> `/develop` -> `/review`.

---

## Work Details

### BLOCK/WARN Hook Separation

Refactored `post-edit-check.sh` into two tiers: BLOCK (immediate halt) and WARN (warning only). BLOCK covers items requiring immediate fixes such as `dart analyze` errors and forbidden pattern usage. WARN covers items for the reviewer to judge, such as TODO count increases and file size overflows.

### Auto-Test Runner Hook Added

Newly added `post-edit-test.sh`. When a modified file has a corresponding test file, it automatically runs only that test. Provides immediate feedback without running the entire test suite every time.

### run-task.sh / run-epic.sh Automation Scripts

`scripts/run-task.sh`: Automatically executes a single Task in Plan -> Develop -> Review order. Commits on APPROVE, halts on REQUEST_CHANGES.
`scripts/run-epic.sh`: Decomposes an Epic into slices and automatically runs each slice through the 3-Role workflow.
