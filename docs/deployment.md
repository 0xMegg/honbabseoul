# Deployment

## Vercel

This project is deployed as a Next.js app.

- Framework Preset: `Next.js`
- Output Directory: leave empty so Vercel uses the Next.js build output

Do not set Output Directory to `dist`; `next build` does not produce a `dist` directory.

## Required Environment

Configure these in Vercel for production and preview:

- `NEXT_PUBLIC_SUPABASE_URL`
- `NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY`
- `SUPABASE_SECRET_KEY`
- `NEXT_PUBLIC_NAVER_MAPS_CLIENT_ID`

Legacy fallback keys are still supported locally but should not be preferred for new Vercel environments:

- `NEXT_PUBLIC_SUPABASE_ANON_KEY`
- `SUPABASE_SERVICE_ROLE_KEY`

## Preview Verification

Every preview deployment should at least pass:

- Vercel build reaches `READY`.
- `/ja` renders the Japanese home surface.
- `/manifest.webmanifest`, `/opengraph-image`, `/robots.txt`, and `/sitemap.xml` are generated by the build.
- Local smoke suite passes with deployed-only specs skipped by default.

Protected Vercel previews need a temporary share URL before browser smoke tests. Use `VERCEL_SHARE_URL` with the protected preview link so Playwright can set the Vercel auth cookie before loading `/ja`.

## Gated Remote Smoke

Remote smoke specs are opt-in because they touch external services:

- `DEPLOYED_BASE_URL` + `VERCEL_SHARE_URL`: deployed read-path smoke for protected previews.
- `RUN_DEPLOYED_UGC_PHOTO_SMOKE=true`: submits a temporary pending row with a tiny PNG, verifies storage/public pending isolation, then cleans up.
- `RUN_DEPLOYED_APPROVAL_SMOKE=true`: submits a temporary row, verifies pending isolation, approves it through the admin client, verifies public visibility, then cleans up.

These specs require Supabase admin envs locally:

- `NEXT_PUBLIC_SUPABASE_URL`
- `NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY`
- `SUPABASE_SECRET_KEY`
