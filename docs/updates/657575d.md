# [P0] fix: write_status() multiline value corruption

## Summary
TASK 인자에 줄바꿈이 포함된 모든 `/task` 또는 `/epic` 실행 시, 2번째 실행부터 status 파일이 깨져 `line N: syntax error near unexpected token '('` 또는 `<text>: command not found` 로 즉시 exit. Developer 역할이 기동조차 못 하던 blocker.

## Commits
- `657575d` fix: kody P0-6 — write_status() multiline value corruption (E11)

## Changes
- `src/scripts/run-task.sh` — `_sanitize_val()` 신규: 줄바꿈/CR → space, multi-space 압축, 120자 이상 truncate. `write_status()` 각 pair 처리 직후 적용
- `src/scripts/run-epic.sh` — `write_epic_status()` 에 동일 패턴 (EPIC_NAME 도 멀티라인 가능성 대비)
- `src/scripts/verify-status-file.sh` (신규) — 회귀 테스트 5 케이스 (single-line / multiline+parens+backticks / 5회 반복 / 파일 라인 수 상한 / 200자 truncate marker). PASS 시 exit 0
- `src/.harness-manifest` — 신규 스크립트 `[managed]` 등록

## Manifest classification
- `[managed]` `scripts/run-task.sh`
- `[managed]` `scripts/run-epic.sh`
- `[managed]` `scripts/verify-status-file.sh`
- `[managed]` `.harness-manifest`

## Why
kody Epic 7 addendum E11/P0-6. `STATUS_FILE` 이 `source` 가능한 `KEY='value'` 포맷인데, `write_status()` 가 `grep -v "^${key}="` 로만 기존 라인 필터 → 멀티라인 single-quoted value 의 2번째 줄 이후가 raw text 로 잔류. 다음 source 시 raw text 가 shell 명령으로 실행 → syntax error.

피드백 원문: `docs/downstream-feedback/kody-2026-04-23.md` §E11 / §P0-6.

## Downstream impact
- 영향 대상: 줄바꿈 포함 markdown TASK 를 쓰는 모든 다운스트림 (권장 패턴)
- 적용 후: 멀티라인 TASK 가 status 파일에서는 단일라인으로 flatten 돼 저장됨 (display/monitor 용). shell 변수 `$TASK` 원본은 그대로 → phase 프롬프트는 멀티라인 유지
- **기존 corrupt 된 status 파일 있다면**: `/tmp/<project>-run/task-status` 삭제 후 재실행 권장
- `TASK_NAME` 이 120자 넘으면 `...(truncated)` 가 붙음 (display only, 기능 영향 없음)
