# [P1] feat: check_harness_version auto-apply (Phase 2)

## Summary
`/task` / `/epic` 실행 시 harness 가 stale 이면 경고만 띄우던 기존 동작을 **자동 apply + 적용 결과 보고** 로 대체. 다운스트림이 수동 `upgrade-harness.sh --apply` 를 실행하지 않아도 다음 task 실행 자체가 최신 상태로 시작됨.

## Why
A/B/C 계획의 C 단계. A (다운스트림 stale 유지), B (`docs/updates/` 시스템) 를 전제로 실제 자동화 로직 투입. 수동 apply 흐름은 다운스트림이 깜빡하면 누락되고, 새 업데이트가 쌓이면 쌓일수록 이슈 진단이 복잡해짐. "task 시작 = 최신 상태 보장" 으로 단순화.

## Downstream impact

### 첫 실행 (기존 다운스트림)
이번 Phase 2 code 자체가 업데이트이므로, 다운스트림의 **현재 run-task.sh 는 여전히 Pre-Phase-2** → 기존 "경고만" 동작 그대로 나옴. 사용자가 **1회 수동** `bash scripts/upgrade-harness.sh --apply` 를 해야 Phase 2 code 가 설치됨.

### 이후 실행
Phase 2 code 설치 후: `/task` / `/epic` 진입 → harness stale 감지 → pending 목록 출력 → 자동 apply → task 진행.

### 출력 예시 (apply 필요 시)
```
  Harness: v4.0.0 (forge 5fdf9ff, built 2026-04-23T13:57:55Z)
  ⚠ Harness out of date: forge 5fdf9ff → 4d02f86
  Pending updates:
    [P1] check_harness_version auto-apply (Phase 2) (4d02f86)
    [P2] harness updates changelog system (8f2cea7)
    [P0] write_status multiline corruption (657575d)
  Auto-applying harness updates...
  ✓ Harness updated to forge 4d02f86
```

### 출력 예시 (최신)
```
  Harness: v4.0.0 (forge 4d02f86, built ...)
  ✓ Harness up-to-date (forge 4d02f86)
```

### 실패 시
`upgrade-harness.sh --apply` 실패 → task 진행 중단 (exit 1). 메시지: "Fix the reported issue or set HARVEST_SKIP_UPDATE_CHECK=1 to proceed with stale harness".
