# Honbabseoul

Honbabseoul is a mobile-first Next.js app for Japanese travelers looking for solo-friendly restaurants in Seoul.

## Product Surface

- `/ja` is the default Japanese MVP route.
- `/ko` is the Korean route.
- Public read path shows only `approved` restaurants.
- UGC submissions are saved as `pending` and require operator approval before appearing on the map.
- Discoverability routes are generated by Next.js: `/manifest.webmanifest`, `/opengraph-image`, `/robots.txt`, and `/sitemap.xml`.

## Operating Layer

Hermes is the active operating layer for this repo.

- Start with `AGENTS.md`.
- Active handoff lives in `.hermes/NEXT.md`.
- Project knowledge lives in `.hermes/wiki/`.
- Decisions and verification live in `.hermes/logs/log.md`.
- Legacy harness material is archived under `.hermes/archive/legacy-harness-2026-05-03/` and is not active workflow authority.

## Development

- Install: `pnpm install`
- Dev: `pnpm dev`
- Lint: `pnpm lint`
- Test: `pnpm test`
- Build: `pnpm build`
- E2E: `pnpm test:e2e`

## Operations

- Deployment runbook: `docs/deployment.md`

- Epic 5 audit: `outputs/reviews/epic-5-audit.md`

Remote smoke tests are gated because they touch deployed previews and Supabase. See `docs/deployment.md` for the required environment variables and flags.
